互联网的工作原理

阮一峰——互联网协议

引导语：全世界数以亿计的计算机，连接在一起，两两通信。深圳的某一网卡送出信号，洛杉矶的另一块网卡居然就收到了，两者实际上根本不知道对方的物理位置，你不觉得这是很神奇的事情吗？

  • 交流的规则：所有连入互联网的计算机交换数据需要遵守的协议
  • 思想：互联网传输为分层传输 每一层有每一层需要遵守的协议
    • 越下面的层，越靠近硬件；越上面的层，越靠近用户
    • 本质：就是将数据一层层进行封装，数据封装时靠近上层的变动完全不涉及下层结构的变动

物理层 Physical Layer

  • 作用：只是负责传送0和1的电信号
  • 媒介：光纤、无线电波、电缆等

链接层 Linked Layer

  • 作用：单纯的0与1电信号没有意义，需要解释，给电信号分组
  • 以太网协议：
    • 出现：每家公司都有自己的电信号分组方式。逐渐地，一种叫做”以太网”的协议，占据了主导地位，一统江湖。
    • 以太网规定：一组电信号构成一个数据包，叫做”帧”（Frame）。每一帧分成两个部分：标头（Head）和数据（Data）。
      • “标头”包含数据包的一些说明项，比如发送者、接受者、数据类型等。
      • “数据”则是数据包的具体内容。
    • 广播：以太网协议规定，必须知道双方的MAC地址，才能将数据广播出去。对方对比标头的MAC地址，符合的才能够接收。
    • 局限：不可能让世界上每一个计算机都来接受这个数据包并判断。互联网是由很多个子网络组成的，你发出的数据包只能在所属的子网络进行广播。（那么如何提高效率？）
    • 问题：如果两台计算机所处的不是同一个子网络，则通过“路由”的方式，将数据路由出去。（类似于：同城有邮递员对着门牌号，不同城，只能放入邮递箱，给个目标地址的邮编），而子网络的地址就是邮编。
    • 解决方案：”网络层”的诞生。它的作用是引进一套新的地址，使得我们能够区分不同的计算机是否属于同一个子网络。这套地址就叫做”网络地址”，简称”网址”。 于是，”网络层”出现以后，每台计算机有了两种地址，一种是MAC地址，另一种是网络地址。两种地址之间没有任何联系，MAC地址是绑定在网卡上的，网络地址则是管理员分配的，它们只是随机组合在一起。

补充：什么是MAC地址？

MAC地址：每一台计算机的网卡出厂都有唯一的编号，成为MAC地址。

以太网协议标头（发送者与接收者的信息标识）：俗称计算机的物理地址（MAC Media Access Control）

ARP（Address Resolution Protocol） 地址解析协议：负责将某个设备IP地址解析，查找该目标设备对应的MAC地址。
ARP攻击：

* 局限性：ARP攻击仅能在以太网（局域网、公司内网）内进行，无法对外网（互联网、非局域网）进行攻击
* ARP攻击原理：就是欺骗、伪造IP/MAC地址的对应关系，可以将A、B两台计算机的交互全部发送至中间人C中，从而实现窃取、修改通信内容

网络层 Network Layer

  • 本质：区分以太网协议中哪些计算机的MAC地址在同一个子网络中
  • 逻辑：两台计算机互通信息，从逻辑上可以推断，必定是先处理网络地址，然后再处理MAC地址。
  • IP协议
    • 规定网络地址的协议叫做IP协议，定义的地址为IP地址。（有时访问一个服务器接口，服务器会去做IP地址过滤，不是白名单的不让访问，为了数据库的安全）
    • 版本：IPv4 IPv6

IPv4

网络地址由32个二进制位组成，即4字节。

习惯上不用二进制标识，而是将每个字节用十进制表示IP地址

0.0.0.0-255.255.255.255

IP地址分成两个部分，前一部分代表网络，后一部分代表主机。比如，IP地址172.16.254.1，假定它的网络部分是 172.16.254，那么主机部分就是1。（一台主机标识唯一一台计算机，就可通过ARP协议解析出MAC地址）

处于同一个子网络的计算机，网络部分必定是相同的。

但是，问题在于如何划分IP地址的网络部分？这就要用到另一个参数”子网掩码”。

子网掩码：表示子网络特征的一个参数。

它在形式上等同于IP地址，也是一个32位二进制数字，它的网络部分全部为1，主机部分全部为0。比如，IP地址172.16.254.1，如果已知网络部分是前24位，主机部分是后8位，那么子网络掩码就是11111111.11111111.11111111.00000000，写成十进制就是255.255.255.0。

如果网络部分是 172.16.254 占了 3 位，那么该子网络最多只能容纳 256 台计算机

IP协议地址作用：

  • 每一台计算机分配地址
  • 确定不同IP地址是否在同一个子网络

根据IP协议发送的数据包就是IP数据包
本质：再一次封装，该层数据标头中，肯定有IP地址

扩充：ARP协议 IP地址–>MAC地址

因为IP数据包是放在以太网数据包里发送的，所以我们必须同时知道两个地址，一个是对方的MAC地址，另一个是对方的IP地址。通常情况下，对方的IP地址是已知的（后文会解释），但是我们不知道它的MAC地址。
